## Quick Setup

### 1. Clone / Open in Visual Studio

Open `EWalletAPI.csproj` in Visual Studio.

### 2. Configure `appsettings.Development.json`

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Port=3306;Database=ewallet_db;User=root;Password=YOUR_PASSWORD;CharSet=utf8mb4;"
  },
  "JwtSettings": {
    "SecretKey": "YOUR_SUPER_SECRET_KEY_MIN_32_CHARS_!@#$",
    "Issuer": "EWalletAPI",
    "Audience": "EWalletUsers",
    "ExpirationInMinutes": 60
  }
}
```

### 3. Run EF Core Migrations

**Package Manager Console** (Visual Studio):
```powershell
Add-Migration InitialCreate -OutputDir Data/Migrations
Update-Database
```

**Or .NET CLI:**
```bash
dotnet ef migrations add InitialCreate --output-dir Data/Migrations
dotnet ef database update
```

### 4. Run the Project

Press **F5** in Visual Studio or:
```bash
dotnet run
```

Swagger UI → `http://localhost:5067/swagger`

## Environment Variables (Production)

In production, override appsettings via environment variables:

| Variable          | Description                    |
|-------------------|-------------------------------|
| `DB_HOST`         | MySQL server hostname          |
| `DB_PORT`         | MySQL port (default 3306)      |
| `DB_NAME`         | Database name                  |
| `DB_USER`         | MySQL username                 |
| `DB_PASSWORD`     | MySQL password                 |
| `JWT_SECRET_KEY`  | JWT signing key (32+ chars)    |
| `JWT_ISSUER`      | Token issuer string            |
| `JWT_AUDIENCE`    | Token audience string          |

Or use the direct `ConnectionStrings__DefaultConnection` env var pattern for ASP.NET Core.

## API Endpoints

### Auth
| Method | Endpoint              | Auth Required |
|--------|-----------------------|---------------|
| POST   | `/api/auth/register`  | No            |
| POST   | `/api/auth/login`     | No            |

### Wallet
| Method | Endpoint                  | Auth Required |
|--------|---------------------------|---------------|
| GET    | `/api/wallet/balance`     | ✅ Bearer JWT  |
| POST   | `/api/wallet/add-money`   | ✅ Bearer JWT  |
| POST   | `/api/wallet/transfer`    | ✅ Bearer JWT  |

### Transactions
| Method | Endpoint              | Auth Required |
|--------|-----------------------|---------------|
| GET    | `/api/transactions`   | ✅ Bearer JWT  |

## Sample Requests

### Register
```http
POST /api/auth/register
Content-Type: application/json

{
  "fullName": "John Doe",
  "email": "john@example.com",
  "password": "Secret@123"
}
```

### Login
```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "john@example.com",
  "password": "Secret@123"
}
```

### Add Money
```http
POST /api/wallet/add-money
Authorization: Bearer {token}
Content-Type: application/json

{
  "amount": 500.00,
  "description": "Salary credit"
}
```

### Transfer
```http
POST /api/wallet/transfer
Authorization: Bearer {token}
Content-Type: application/json

{
  "receiverEmail": "jane@example.com",
  "amount": 150.00,
  "description": "Splitting dinner bill"
}
```

### Transaction History (Paginated)
```http
GET /api/transactions?page=1&pageSize=10
Authorization: Bearer {token}
```

## Security Features

- **BCrypt** password hashing (work factor 11)
- **JWT** with HMAC-SHA256, configurable expiry
- **[Authorize]** attribute on all wallet/transaction endpoints
- **Database transactions** for atomic debit/credit (EF Core `BeginTransactionAsync`)
- **Deadlock prevention** via consistent wallet-lock ordering
- **Global exception middleware** — never leaks stack traces in production
- **Input validation** via DataAnnotations + ModelState checks
- **Self-transfer prevention** guard
- **Insufficient balance** check before any debit
